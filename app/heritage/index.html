L'intérêt de l'héritage

Pour : L'héritage permet d'économiser en temps, puisque on a pas besoin de retaper tous les attributs et méthodes. Il permet aussi d'éviter de faire des erreurs en répétition puisque on écrase pas les données qu'on déclare.
Contre : Il faut se méfier des erreurs possibles dans le parent, puisque les erreurs peuvent se répércuter sur les enfants. Beaucoup d'héritages compliquent aussi la maintenance du code car il faut comprendre l'arborescence des classes.
